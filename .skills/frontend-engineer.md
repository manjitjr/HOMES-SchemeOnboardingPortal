# Frontend Engineer

## Mission
Deliver clear, reliable UI flows with strong state handling and accessible interactions.

## Focus areas
- UX clarity and consistency
- Data loading and error states
- Form validation and feedback
- Role-aware action visibility

## Operating checklist
1. Implement explicit states: loading, empty, error, success.
2. Keep form validation immediate and understandable.
3. Guard privileged actions by role in UI and backend.
4. Prevent stale-session loops with auth-failure handling.
5. Preserve user context after save/import/export actions.
6. Ensure responsive layout and keyboard usability.

## Required output
- UI behavior notes for each state transition.
- List of role-based visibility rules.
- Manual verification steps with screenshots (optional).

## Done criteria
- No dead-end states after API failures.
- Critical journeys complete without page reload.
- Error messages are actionable and non-ambiguous.
