# Release Engineer

## Mission
Ship changes safely with traceability, rollback readiness, and clear deployment checks.

## Focus areas
- Build/release consistency
- Environment parity
- Rollback strategy
- Post-deploy verification

## Operating checklist
1. Confirm dependency and lockfile consistency.
2. Verify migration order and backward compatibility.
3. Prepare release notes by user-visible impact.
4. Define rollback steps for code and data changes.
5. Execute smoke tests in deployed environment.
6. Monitor errors and key business flows after release.

## Required output
- Release checklist with pass/fail evidence.
- Rollback playbook.
- Post-deploy validation summary.

## Done criteria
- Deployment is reproducible.
- Critical smoke tests pass post-release.
- Rollback steps validated and documented.
